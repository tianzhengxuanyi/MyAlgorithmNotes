## Node.js 的事件循环机制

事件循环 = 阶段轮询 + 回调调度 + 微任务优先，让单线程 Node.js 高效处理高并发 I/O，核心是不阻塞主线程、异步回调排队执行。

### 事件循环机制
事件循环机制是Node.js的核心机制，它允许Node.js非阻塞地执行I/O操作。

事件循环的各个阶段：
1. Timers(定时器阶段)：执行setTimeout和setInterval回调函数。
2. Pending Callbacks（待定回调）：处理上一轮延迟到本轮的 I/O 系统回调（如 TCP 错误、fs 错误）。
3. idle, prepare(空闲阶段)：Nodejs内部使用，不对用户公开。
4. poll(轮询阶段)：
   - 取新 I/O 事件（网络、文件），执行绝大多数 I/O 回调（fs、http 等）。
   - 队列空时：
      - 有定时器 → 不阻塞，直接进下一阶段
      - 无定时器 → 阻塞等新 I/O（直到有事件或超时）。
5. check(检查阶段)：专门用来执行setImmediate回调函数，这是一个立刻执行的机制，不会等到下一个循环。
6. close callbacks(关闭回调阶段)：执行close事件的回调函数，如socket.on('close', ...)、http.Server#close()、net.Socket#end()等。

### 微任务
**微任务（Microtask）** 不在事件循环阶段里，**每阶段执行完后、下一阶段开始前，会清空所有微任务**。
微任务（优先级最高）：
  - process.nextTick（Node 专属，比 Promise 还高）
  - Promise.then / catch / finally
  - queueMicrotask

宏任务（事件循环各阶段队列）：
  - Timers：setTimeout / setInterval
  - Poll：I/O 回调
  - Check：setImmediate
  - Close：close 回调

### 常见易错点
1. setTimeout 与 setImmediate 谁先执行？
- I/O 回调里：setImmediate 一定先于 setTimeout（先 Check 再 Timers）。
- 全局直接写：不确定（取决于事件循环启动时的状态，可能 setTimeout 先）。
```js
// 全局：顺序不确定
setTimeout(() => console.log('timeout'), 0);
setImmediate(() => console.log('immediate'));

// I/O 回调里：immediate 一定先
fs.readFile(__filename, () => {
  setTimeout(() => console.log('timeout'), 0);
  setImmediate(() => console.log('immediate'));
});
```

2. process.nextTick 会 “插队”
它在当前操作结束后、事件循环下一阶段前立即执行，优先级最高，过多会饿死 I/O 回调。
```js
setImmediate(() => console.log('immediate'));
process.nextTick(() => console.log('nextTick'));
// 输出：nextTick → immediate
```
## Node.js与浏览器事件循环的异同

### 回答重点
- 相似点：均基于**事件驱动机制**，用于管理异步操作
- 核心差异：实现细节和主要用途不同
  1. Node.js事件循环：服务端场景，主要处理**I/O操作**（文件系统读写、网络请求等）
  2. 浏览器事件循环：客户端场景，处理用户交互、DOM操作、HTTP请求等

### 扩展知识
#### Node.js事件循环
- 底层实现：基于跨平台底层库**libuv**，用于处理异步I/O
- 阶段划分：包含6个阶段，每个阶段执行特定类型回调（定时器、I/O任务、check阶段、close callback等）
- 微任务处理：每个阶段结束后处理微任务队列（如Promise）

#### 浏览器事件循环
- 底层实现：由浏览器厂商实现，通常基于HTML5规范
- 任务队列：所有异步任务排入单一队列，按先进先出顺序执行
- UI渲染阶段：包含专门的UI更新阶段，进行DOM渲染和页面重绘
- 微任务处理：每个宏任务（如setTimeout）执行后处理微任务（如Promise.then），保证微任务优先级更高

### 具体区别
1. **UI渲染阶段**：浏览器独有，每个宏任务后执行；Node.js无此阶段（服务端无需处理UI）
2. **任务调度**：Node.js按不同阶段调度任务；浏览器主要分为宏任务和微任务两种
3. **运行环境**：Node.js面向服务端，侧重文件系统、网络等异步操作；浏览器面向客户端，侧重用户交互和页面渲染


## Node.js 常见全局对象（适合背诵版）
### 核心全局对象
1. **global**：全局命名空间，类比浏览器`window`，属性方法可任意访问，开发中避免新增属性防污染
2. **process**：进程信息与控制中心（EventEmitter实例）
   - **核心属性**：
     - `argv`：命令行参数数组（Node路径+脚本路径+参数）
     - `env`：Shell环境变量对象
     - `pid`：当前进程ID
     - `cwd()`：返回当前工作目录
   - **常用方法**：
     - `exit([code])`：终止进程，可选退出码
     - `chdir(directory)`：切换工作目录
     - `nextTick(callback)`：添加到下一个tick队列（微任务）
     - `on(event, listener)`：注册事件监听器（exit/uncaughtException/SIGINT等）
   - **应用场景**：
     - 监听进程退出，执行资源清理
     - 读取命令行参数和环境变量
     - 切换/获取工作目录、获取进程ID
     - 实现高效异步操作（nextTick）
3. **console**：调试输出工具，提供`log/error/warn`等方法
4. **定时器相关**：
   - `setTimeout/clearTimeout`：单次延迟执行回调，可取消
   - `setInterval/clearInterval`：周期性重复执行回调，可取消
   - `setImmediate/clearImmediate`：事件循环结束后执行回调，可取消
5. **Buffer**：二进制数据处理工具，用于文件/网络I/O，操作字节缓冲区
6. **模块相关**：
   - `require`：导入其他模块的函数
   - `module`：表示当前模块，文件即模块，可导出内容供其他模块使用

### 注意事项
这些对象全局可用，但在模块中属于模块作用域变量，非严格意义的全局变量（模块内定义的变量不会自动成为全局变量）

## Node.js 垃圾回收机制（适合背诵版）
### 核心概念
- **基于V8引擎**：Node.js的GC本质是V8引擎的垃圾回收机制
- **标记-清除算法**：遍历对象图标记可达对象，清除未标记对象释放内存
- **自动回收**：默认自动进行，无需手动干预

### 手动触发GC
1. **启动时暴露GC**：使用`--expose-gc`标志启动Node.js
   ```bash
   node --expose-gc your_script.js
   ```
2. **代码中调用**：检查`global.gc`是否存在，调用触发回收
   ```js
   if (global.gc) {
     global.gc(); // 手动触发垃圾回收
   }
   ```

## Node.js 内存管理机制与内存泄漏预防

### 一、内存管理机制核心
- 依赖**V8引擎**（Google开发的高效JS引擎），负责JavaScript内存分配与垃圾回收
- 内存分为4部分：
  - **栈内存**：存储局部变量、函数调用
  - **堆内存**：存储对象、字符串等复杂数据结构
  - 原生内存、内核内存
- 垃圾回收（GC）由V8完成，自动回收不再使用的内存

### 二、避免内存泄漏和溢出的措施
1. **及时释放资源**：使用后关闭数据库连接、文件句柄等资源
2. **合理使用全局变量**：尽量避免全局变量，防止内存堆积
3. **规避闭包泄漏**：注意闭包对外部变量的引用，避免内存无法回收
4. **使用诊断工具**：利用Node.js内存分析工具（如`memory-leak-detector`、`heapdump`）
5. **优化代码逻辑**：检查长时间运行的循环、递归，避免内存过度占用

### 三、扩展知识（进阶）
#### 1. 对象池技术
- 设计模式：池化资源，减少频繁内存分配/回收，降低GC负担（适用于高频操作场景）

#### 2. 内存分析工具
- **process.memoryUsage()**：获取当前进程内存使用情况
- **heapdump**：生成堆快照，分析内存使用情况
- **node-inspect**：运行时检查对象内存占用
- **Chrome DevTools**：连接Node.js程序，做详细内存分析与调试

#### 3. 垃圾回收算法
- 了解核心算法：**标记-清除**、**标记-整理**、**分代回收**，帮助优化代码性能

#### 4. 内存限制
- Node.js默认内存限制约1.5GB
- 可通过运行参数调整：`--max-old-space-size`（适配不同应用需求）

### 扩展知识（背诵重点）
1. **V8优化策略**：
   - **增量标记**：将标记工作分成小块，避免长时间停顿
   - **并发回收**：清除阶段多线程并行，减少主线程阻塞
2. **内存分区**：
   - **新生代**：新对象存放区，GC频繁但速度快
   - **老生代**：存活对象转移区，GC较少但耗时较长
3. **内存优化建议**：
   - 减少全局变量使用，及时释放内存
   - 使用stream流处理，避免一次性加载大数据集
   - 优化数据结构，选择合适算法
   - 低负载时手动触发GC清理内存
4. **监控工具**：
   - Node.js Inspector + Chrome开发者工具：调试监控内存
   - heapdump/node-memwatch：生成堆快照、监控内存泄漏

## Node.js中require方法的文件查找策略是怎样的？

1. 检查内置模块： 如果模块名是Node.js的内置模块，直接返回模块对象。
2. 检查缓存： 如果模块名是已加载的模块，直接返回缓存中的模块对象。
3. 解析文件路径: 如果提供的是一个相对路径或绝对路径，根据路径定位到具体文件。
4. 文件扩展名尝试： 如果文件扩展名为空，尝试添加.js、.json、.node等扩展名。
5. 解析目录：如果传递的是一个目录路径
   - 查找该目录下的package.json文件，读取package.json文件中的main字段，根据main字段定位到模块入口文件。
   - 如果package.json文件不存在或main字段为空，尝试查找该目录下的index.js、index.json、index.node等文件。
6. 查找node_modules目录： 从当前目录依次递归向上查找node_modules目录，直到找到或到达根目录。

## 什么是Stream流？

Node.js 中的 Stream 流是一种用于处理数据的机制，它允许你以流式的方式读取和写入数据，而不会占用大量的内存。
Stream 流分为四种类型：可读流（Readable）、可写流（Writable）、双流（Duplex）、转换流（Transform）

```js
const fs = require("fs");
const { Transform } = require("stream");

// 可读流
const readableStream = fs.createReadStream("./test.txt");
// 可写流
const writableStream = fs.createWriteStream("./test2.txt");
// 将可读流转换为大写流
const upperCaseTransform = new Transform({
  transform(chunk, encoding, callback) {
    this.push(chunk.toString().toUpperCase());
    callback(null, chunk.toString().toUpperCase());
  },
});

// 管道流
readableStream.pipe(upperCaseTransform).pipe(writableStream);
```

## 什么是集群？如何在Node中实现集群和负载均衡？

集群是指在Node.js中运行多个进程，每个进程都执行任务，以提高性能和可扩展性。

1. 加载cluster等模块
2. 使用cluster.isMaster判断是否为主进程
3. 如果是主进程，使用cluster.fork()创建子进程
4. 如果是子进程，创建HTTP服务器，并监听指定端口，处理请求和响应
5. 监听exit事件，当子进程退出时，创建新的子进程

```js
const cluster = require("cluster");
const http = require("http");
const numCPUs = require("os").cpus().length;

if (cluster.isMaster) {
  console.log(`Master ${process.pid} is running`);

  // Fork workers.
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }

  cluster.on("exit", (worker, code, signal) => {
    console.log(`Worker ${worker.process.pid} died`);
    console.log("Starting a new worker");
    cluster.fork();
  });
} else {
  // Workers can share any TCP connection
  // In this case, it is an HTTP server
  http
    .createServer((req, res) => {
      res.writeHead(200);
      res.end("Hello World\n");
    })
    .listen(8000);

  console.log(`Worker ${process.pid} started`);
}
```

## Node.js中exec、execFile、spawn 和 fork的区别

在 Node.js 中，exec、execFile、spawn 和 fork 都是用于创建子进程的，但它们有不同的功能和使用场景：
- exec：用于执行 shell 命令并将命令输出作为回调函数的参数返回。适用于需要执行短命令并将结果作为字符串处理的场景。
- execFile：用于直接执行特定文件（例如二进制文件或脚本文件），不创建新的 shell。适用于不需要 shell 特性的场景，如运行已知的可执行文件。
- spawn：用于启动新的子进程并与其进行实时的输入输出流（stdin、stdout、stderr）交互。适合处理长时间运行的进程或需要进行数据流处理的任务。
- fork：是专门用于衍生新的 Node.js 进程的，通常用于创建工作进程以分担主进程的工作负载。适用于多进程架构的场景，如实现并行任务处理。


## Node.js 如何处理多线程或多进程？

- child_process模块用于创建子进程，执行系统命令及脚本。
  - exec方法用于执行系统命令，返回命令的输出。
  - spawn方法用于创建子进程，执行系统命令。
  - fork方法用于创建子进程，执行Node.js脚本。
- worker_threads模块用于创建多线程，执行Node.js代码。

**exec**

```js
const { exec } = require("child_process");

exec("ls -la", (error, stdout, stderr) => {
  if (error) {
    console.error(`exec error: ${error}`);
    return;
  }
  console.log(`stdout: ${stdout}`);
  console.error(`stderr: ${stderr}`);
});
```

**spawn**

```js
const { spawn } = require("child_process");

const ls = spawn("ls", ["-la"]);

ls.stdout.on("data", (data) => {
  console.log(`stdout: ${data}`);
});

ls.stderr.on("data", (data) => {
  console.error(`stderr: ${data}`);
});

ls.on("close", (code) => {
  console.log(`child process exited with code ${code}`);
});
```

**fork**

```js
const { fork } = require("child_process");

const child = fork("child_script.js");

child.on("message", (msg) => {
  console.log("Message from child", msg);
});

child.send({ hello: "world" });
```

**worker_threads**：用于创建多线程，执行Node.js代码。提供了较小的开销，并且可以共享主线程的内存，快速地交换数据。

```js
const { Worker, isMainThread, parentPort } = require("worker_threads");

if (isMainThread) {
  // 主线程
  const worker = new Worker(__filename);

  worker.on("message", (msg) => {
    console.log("Message from worker:", msg);
  });

  worker.postMessage("Hello, worker!");
} else {
  // Worker 线程
  parentPort.on("message", (msg) => {
    console.log("Message from parent:", msg);
    parentPort.postMessage("Hello, parent!");
  });
}
```

## 如何在node中实现文件的压缩和解压？

使用内置的zlib模块实现文件的压缩和解压。

**文件压缩：**

```js
const fs = require('fs');
const zlib = require('zlib');

const inputFile = 'input.txt';
const outputFile = 'input.txt.gz';

const gzip = zlib.createGzip();
const input = fs.createReadStream(inputFile);
const output = fs.createWriteStream(outputFile);

input.pipe(gzip).pipe(output);

output.on('finish', () => {
    console.log(`File successfully compressed to ${outputFile}`);
});
```

**文件解压：**
```js
const fs = require('fs');
const zlib = require('zlib');

const inputFile = 'input.txt.gz';
const outputFile = 'output.txt';

const gunzip = zlib.createGunzip();
const input = fs.createReadStream(inputFile);
const output = fs.createWriteStream(outputFile);

input.pipe(gunzip).pipe(output);

output.on('finish', () => {
    console.log(`File successfully decompressed to ${outputFile}`);
});
```

## 在node中如何处理文件上传和下载？

通过express和multer模块实现文件上传和下载。

```js
const express = require('express');
const multer = require('multer');
const path = require('path');

const app = express();
const PORT = process.env.PORT || 3000;

// 配置 multer
const storage = multer.diskStorage({
  // 上传文件存储目录
  destination: (req, file, cb) => {
    cb(null, 'uploads/');
  },
  // 上传文件存储文件名
  filename: (req, file, cb) => {
    cb(null, `${Date.now()}-${file.originalname}`);
  },
});
const upload = multer({ 
  storage,
  // 限制上传文件类型
  fileFilter: (req, file, cb) => {
    const fileTypes = /jpeg|jpg|png|gif/;
    const mimeType = fileTypes.test(file.mimetype);
    const extname = fileTypes.test(path.extname(file.originalname).toLowerCase());

    if (mimeType && extname) {
      return cb(null, true);
    }
    cb('Error: File type not supported!');
  },
  // 限制上传文件大小
  limits: { fileSize: 2 * 1024 * 1024 },  // 2MB
});

// 上传接口
// upload.single('file') 中间件 用于处理单个文件上传
app.post('/upload', upload.single('file'), (req, res) => {
  res.send('File uploaded successfully.');
});

// 下载接口
app.get('/download/:filename', (req, res) => {
  const filePath = path.join(__dirname, 'uploads', req.params.filename);
  res.download(filePath);
});

// 流式下载接口
app.get('/download/:filename', (req, res) => {
  const filePath = path.join(__dirname, 'uploads', req.params.filename);
  const fileStream = fs.createReadStream(filePath);
  fileStream.pipe(res);
});


app.listen(PORT, () => {
  console.log(`Server is running on port ${PORT}`);
});

```

## 如何在Node.js中实现数据的缓存，提高性能？

- 内存缓存：使用node-cache、memory-cache
- 分布式缓存：使用Redis

**内存缓存：node-cache**
```js
const NodeCache = require('node-cache');
const myCache = new NodeCache();

// 设置缓存数据
myCache.set("key", "value", 10000); // 数据10秒后过期

// 获取缓存数据
const value = myCache.get("key");
console.log(value); // 输出 'value'
```

**内存缓存：memory-cache**
```js
const cache = require('memory-cache');

// 设置缓存数据
cache.put('key', 'value', 10000); // 数据10秒后过期

// 获取缓存数据
const value = cache.get('key');
console.log(value); // 输出 'value'
```

**分布式缓存：Redis**
```js
const redis = require('redis');
const client = redis.createClient();

client.on('error', (err) => {
  console.log('Error ' + err);
});

// 连接 Redis 服务器
client.connect();

// 设置缓存数据，10秒后过期
client.set('key', 'value', 'EX', 10);

// 获取缓存数据
client.get('key', (err, value) => {
  if (err) throw err;
  console.log(value); // 输出 'value'（如果数据未过期）
});
```

## Node.js中的守护进程如何实现的？

使用child_process模块创建守护进程，通过将父进程退出并且保持子进程运行，实现守护进程的功能。
1. 使用 spawn 方法创建子进程
2. 使用 detached 选项使子进程独立于父进程运行
3. 将输出重定向到文件，以便子进程不依赖终端
4. 退出父进程，使子进程成为孤儿进程

```js
const { spawn } = require('child_process');
const fs = require('fs');

// 定义守护进程的基本逻辑
function startDaemon() {
  const out = fs.openSync('./out.log', 'a');
  const err = fs.openSync('./err.log', 'a');

  const child = spawn(process.argv[0], ['child.js'], {
    detached: true,
    stdio: ['ignore', out, err]
  });

  // 让子进程不依赖父进程关闭
  child.unref();

  console.log(`Daemon process started with PID: ${child.pid}`);
}

// 如果当前进程是父进程，则启动守护进程
if (process.argv[2] !== 'daemon') {
  startDaemon();
  process.exit();
}

// 子进程的逻辑（保存在 child.js 文件中）
if (process.argv[2] === 'daemon') {
  setInterval(() => {
    console.log(`Daemon running with PID: ${process.pid}`);
  }, 1000);
}
```

## 如何在Node.js中处理文件系统的监控？

使用fs.watch或者fs.watchFile方法监控文件系统的变化。fs.watch适合监听目录的变化，fs.watchFile适合监听单个文件的变化。

**fs.watch：**
1. 基于事件驱动的文件系统监控，可以监听文件或目录的变化事件，包括rename和change事件
2. 支持控制是否递归监听目录

```js
const fs = require('fs');

fs.watch('path/to/file/or/directory', (eventType, filename) => {
  if (filename) {
    console.log(`${filename} file Changed due to ${eventType}`);
  }
});
```

**fs.watchFile：**
1. 基于轮询的文件系统监控，可以监听单个文件的变化事件
2. 支持设置轮询间隔时间，默认1000毫秒

```js
const fs = require('fs');

fs.watchFile('path/to/file', { interval: 500 }, (curr, prev) => {
  console.log(`File modified at: ${curr.mtime}`);
});
```

## 如何在Node.js中处理和解析二进制数据？

用Buffer模块处理二进制数据。
1. 创建Buffer对象
2. 将数据写入Buffer对象
3. 从Buffer对象读取数据
4. 操作Buffer，如拼接、切分、转换等

- Buffer.alloc(size, encoding, offset, length) 创建一个指定大小的Buffer对象
- Buffer.allocUnsafe(size, encoding, offset, length) 创建一个未初始化的Buffer对象
- Buffer.from(data, encoding, offset, length) 从其他数据创建Buffer对象
- Buffer.concat(buffers, totalLength) 拼接多个Buffer对象
- buf1.toString(encoding) 将Buffer对象转换为字符串
- buf1.write(data, offset, length, encoding) 将字符串写入Buffer对象
- buf1.slice(start, end) 切分Buffer对象
- buf1.compare(buf2) 比较两个Buffer对象
- buf1.fill(data, start, end) 填充Buffer对象

```js
// 1. 创建一个 Buffer
let buf1 = Buffer.alloc(10);  // 分配一个大小为 10 字节的 Buffer
let buf2 = Buffer.from('Hello, World!', 'utf8');  // 从字符串创建 Buffer

// 2. 将数据写入 Buffer
buf1.write('Hello');

// 3. 从 Buffer 中读取数据
let data = buf2.toString('utf8');
console.log(data);  // 输出: "Hello, World!"

// 4. 操作 Buffer
let buf3 = Buffer.concat([buf1, buf2]);
console.log(buf3.toString('utf8'));  // 输出: "HelloHello, World!"
```

## 如何在Node.js中使用Redis实现数据缓存

```js
const redis = require('redis');
const client = redis.createClient();

client.on('error', (err) => {
    console.error('Redis error:', err);
});

client.on('connect', () => {
    console.log('Connected to Redis');
});

const fetchData = async (key) => {
    // 尝试从 Redis 获取数据
    client.get(key, async (err, data) => {
        if (err) throw err;

        if (data) {
            // 如果缓存命中，返回缓存数据
            console.log('Cache hit');
            return JSON.parse(data);
        } else {
            // 如果缓存未命中，从数据库获取数据（模拟）
            console.log('Cache miss');
            const freshData = await getFromDatabase(key); // getFromDatabase() 是模拟的数据库查询函数
            
            // 将新获取的数据放入缓存，设置过期时间为600秒
            client.setex(key, 600, JSON.stringify(freshData));
            
            return freshData;
        }
    });
};

const getFromDatabase = async (key) => {
    // 模拟从数据库获取数据的延迟
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve({ value: `Data for ${key}` });
        }, 1000);
    });
};

// 使用fetchData函数
(fetchData("testKey")).then(data => {
    console.log(data);
});
```

## 在Node.js中如何使用JWT实现用户认证

1. 生成令牌：当用户登录或注册时，生成一个JWT令牌，并将其发送给客户端。
2. 验证令牌：在客户端每次请求时，将令牌包含在请求头中。服务器收到请求后，验证令牌的签名和过期时间。
3. 中间件处理：编写中间件函数，用于在请求处理链中验证和解析令牌。

```js
const express = require('express');
const jwt = require('jsonwebtoken');
const app = express();
const secretKey = 'your_secret_key';

// 用户授权后生成JWT
app.post('/login', (req, res) => {
  const user = { id: 1, username: 'john' }; // 模拟的用户数据
  const token = jwt.sign(user, secretKey);
  res.json({ token });
});

// 中间件用于验证JWT
const authenticateJWT = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (token) {
    jwt.verify(token, secretKey, (err, user) => {
      if (err) {
        return res.sendStatus(403); // 令牌无效
      }
      req.user = user; // 将用户信息附加到请求对象上
      next();
    });
  } else {
    res.sendStatus(401); // Token 未提供
  }
};

// 受保护的路由
app.get('/protected', authenticateJWT, (req, res) => {
  res.json({ message: "You are authorized!", user: req.user });
});

app.listen(3000, () => {
  console.log('Server started on port 3000');
});

```

## Node.js 中如何实现自定义的事件发射器（EventEmitter）？

通过继承内置的 EventEmitter 类来实现。

```js
const EventEmitter = require('events');

class MyEmitter extends EventEmitter {}

const myEmitter = new MyEmitter();

// 监听事件
myEmitter.on('event', () => {
  console.log('一个事件被触发！');
});

// 发射事件
myEmitter.emit('event');

```

1. 一次性事件监听器
```js
myEmitter.once('event', () => {
  console.log('第一次也是唯一一次触发事件！');
});

```

2. 移除事件监听器
```js
const callback = () => {
  console.log('这不会被打印');
};

myEmitter.on('event', callback);

// 移除监听器
myEmitter.removeListener('event', callback);

// 事件触发，但没有任何效果
myEmitter.emit('event');

```

3. 设置最大监听器数量
```js
myEmitter.setMaxListeners(20);
```

4. 获取某事件所有监听器
```js
const listeners = myEmitter.listeners('event');
console.log('监听器列表：', listeners);
```

5. 事件监听器传递参数
```js
myEmitter.on('eventWithArgs', (arg1, arg2) => {
  console.log(`参数1: ${arg1}, 参数2: ${arg2}`);
});

myEmitter.emit('eventWithArgs', '参数值1', '参数值2');
```

6. 事件监听器用于异步操作
```js
const fs = require('fs');

class FileReader extends EventEmitter {
  readFile(filename) {
    fs.readFile(filename, (err, data) => {
      if (err) {
        this.emit('error', err);
      } else {
        this.emit('data', data);
      }
    });
  }
}

const fileReader = new FileReader();

fileReader.on('data', (data) => {
  console.log('读取到文件内容：', data.toString());
});

fileReader.on('error', (err) => {
  console.error('读取文件错误：', err);
});

fileReader.readFile('path/to/file.txt');
```

## 什么是BFF？如何使用Node.js实现？

BFF（Backend for Frontend）是一种架构模式，一般指的是在前端与后端之间加增加一个中间层，负责处理前端应用的业务逻辑。

BFF的优势：
1. 服务端对数据展示服务进行解耦，展示服务由独立的BFF端提供，服务端可以聚焦于业务处理。
2. 多端展示或者多业务展示时，对与数据获取有更好的灵活性，避免数据冗余造成消耗服务端资源。
3. 对于复杂的前端展示，将数据获取和组装的负责逻辑在BFF端执行，降低前端处理的复杂度，提高前端页面响应效率。
4. 部分展示业务，可以抽象出来利用BFF实现，对于服务端实现接口复用。
5. 降低多端业务的耦合性，避免不同端业务开发互相影响。
6. 其他优势，包括数据缓存，接口安全校验等。


使用express或koa框架实现BFF。
```js
const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(port, () => {
  console.log(`Server is running at http://localhost:${port}`);
});
```
