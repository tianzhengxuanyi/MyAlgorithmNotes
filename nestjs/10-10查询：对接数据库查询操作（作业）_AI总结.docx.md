# NestJS 数据库查询：联合查询、分页与条件筛选

## 一、核心概念回顾

### 1. 联合查询方案

TypeORM 提供两种联合查询写法：

#### 写法一：relations（推荐）

```typescript
const users = await this.userRepository.find({
  relations: { profile: true, role: true }
});
```

- 直接声明关联属性名
- 简洁直观，适用于大多数场景

#### 写法二：join（显式关联）

```typescript
const users = await this.userRepository.createQueryBuilder('user')
  .leftJoinAndSelect('user.profile', 'profile')
  .leftJoinAndSelect('user.role', 'role')
  .getMany();
```

- 支持更复杂的关联逻辑
- 适用于需要自定义关联条件的场景

### 2. 分页参数映射

| TypeORM 参数 | SQL 关键字 | 计算公式                     |
|--------------|------------|------------------------------|
| `skip`       | `OFFSET`   | `(page - 1) * limit`         |
| `take`       | `LIMIT`    | `limit`（每页显示条数）      |

## 二、代码实现详解

### 1. Repository 层实现

```typescript
// user.service.ts
async getUsers(query: GetUserDto) {
  const { page = 1, limit = 10, username, gender, roleId } = query;
  
  // 计算分页参数
  const take = parseInt(limit as string);
  const skip = (parseInt(page as string) - 1) * take;
  
  // 构建查询条件
  const where: any = {};
  if (username) where.username = username;
  if (gender) where.profile = { gender };
  if (roleId) where.role = { id: roleId };
  
  // 执行查询
  const users = await this.userRepository.find({
    where,
    relations: { profile: true, role: true },
    skip,
    take,
    select: { id: true, username: true, profile: { id: true, gender: true } }
  });
  
  return users;
}
```

### 2. Controller 层实现

```typescript
// user.controller.ts
import { Get, Query } from '@nestjs/common';
import { GetUserDto } from './dto/get-user.dto';

@Get()
getUsers(@Query() query: GetUserDto) {
  return this.userService.getUsers(query);
}
```

## 七、常见陷阱与最佳实践

### 常见陷阱

1. **参数类型错误**：未将 string 类型的 page/limit 转换为 number
2. **关联表条件写法错误**：未正确嵌套关联表条件
3. **N+1 查询问题**：未使用 relations 或 join 导致多次查询

### 最佳实践

1. **使用类型转换**：始终将 page/limit 转换为 number 类型
2. **优先使用 relations**：简洁直观，适用于大多数场景
3. **使用 select 限制字段**：避免返回敏感信息（如 password）
