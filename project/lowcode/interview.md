## 二、架构与原理
- 低代码平台的整体架构一般包含哪些模块？
  - 画布编辑器
  - 组件库
  - 协议 / DSL
  - 渲染引擎
  - 数据源 / 请求封装
  - 权限 / 路由 / 打包发布
- 低代码的核心数据流是什么？
  - 配置 (JSON Schema) → 渲染 → 事件 / 逻辑 → 重新渲染
- 什么是DSL / 协议设计？为什么低代码必须有协议？
- 低代码的渲染模式有哪些？
  - 实时渲染
  - 编译渲染（生成代码）
  - 混合模式
- 低代码编辑器的撤销 / 重做怎么实现？
  - 命令模式 Command Pattern
  - 保存每一步操作历史
- 低代码如何实现组件拖拽？
  - HTML5 Drag & Drop
  - 基于坐标计算
  - 虚拟列表 / 画布优化

## 三、组件系统设计
- 低代码组件规范应该包含哪些字段？

- 低代码组件的生命周期如何管理？
- 如何实现组件嵌套、布局组件（Grid、Flex、Card、Tabs）？
- 组件通信方式：
  - 父 ↔ 子
  - 全局状态 / 页面上下文
  - 事件总线
