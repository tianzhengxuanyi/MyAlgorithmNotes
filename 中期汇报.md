## 1、试用期工作总结；

### 1.1、IOT规则引擎 V2.0

**概述：** 基于IOT平台提供的原始点位数据，将分散在各平台的业务规则抽象注册成算子，以流程化驱动，统一进行逻辑处理并推送结果。

**架构：** 点位设备收集数据 -> 网关 -> IOT平台 -> IOT规则引擎

**目的：** 原先各个平台系统中需要自己对业务中遇到的规则场景进行编码，且不同平台可能会有重复的规则逻辑。规则引擎将这些规则逻辑的设计集成，统一逻辑处理并推送结果；实现规则的统一化管理和解耦；

**业务流程：** 外部接口、数据源配置 -> 导入点位 -> 算子注册 -> 规则流

#### 我参与的模块
  **算子注册**
  - 算子注册列表的代码改造和扩展；
    - vue2选项式组件升级成setup+TS风格；
    - 将Mixin改用为vue hook使用；
  - 左侧子树的改造；
    - 添加搜索功能；
    - hover显示下拉菜单，实现树节点的添加、删除和修改；
  - 算子基本信息、属性、出入参页面；
    - 算子属性表格的排序拖动；
      - 将后端返回的树形数据展平和改造，添加parent字段记录子级的父级元素；
      - sortable.js实现拖动排序，并禁止越级交换；
    - 算子属性、出入参表格中表单的逐行校验；
      - ElementUI表单里嵌套了表格，表格内每行能进行新增、编辑、删除等操作，同时能针对该行的字段进行校验；
    - 图标上传组件的改造；
      - 问题：用户进行上传或删除图标操作后点击取消按钮无法复原原先状态。
      - 解决方案：用户进行上传或删除操作，只对组件的fileList操作，当用户点击保存按钮再根据判断进行；
    - 页面代码的抽离，将基本信息、属性、出入参抽离封装成组件，减少文件代码量；


  **过程点位**
  - 过程点位列表的编码导入、编辑和查询；
  - 过程点位的溯源；

  **规则流（部分）**
  - cron表达式生成组件；
    - 业务需求：生成的规则需要定时去触发，用户可以选择时间频率和corn表达式两种触发模式。之前用户需要借助网上的corn表达式生成工具生成表达式再填入表单中，操作繁琐并且部分用户可能不了解cron表单表达式的规则和作用。
    - 解决思路：github中搜索cron表达式库，对现有的开源库进行改造，添加cron表达式的反解析，展示最近5条执行时间，并结果自动填入表单。


  - 代码生成组件；需求：
    - 算子、常数和运算符的拖拽（vue3 draggable.next插件）
    - 公式的解析和展示
    - 生成公式的常见合法性校验
  - 左侧算子的分组展示；
  - 右侧属性面板的属性改造，如IOT查询算子属性配置-接口改为区域
  - 点位历史、追踪页面、规则警报结果和执行日志页面迁移
